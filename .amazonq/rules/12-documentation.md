# Documentation Standards

## README

- Keep README.md updated with setup instructions
- Include prerequisites and dependencies
- Document environment variables
- Provide troubleshooting tips
- Add contribution guidelines

## API Documentation

- Document API endpoints with request/response examples
- Use OpenAPI/Swagger when possible
- Include error responses
- Document authentication requirements
- Provide example curl commands

## Component Props

- Use TypeScript interfaces for self-documenting props
- Add JSDoc comments for complex props
- Document default values
- Provide usage examples

## Changelogs

- Maintain CHANGELOG.md for significant changes
- Follow Keep a Changelog format
- Group changes by type (Added, Changed, Fixed, Removed)
- Include version numbers and dates

## Architecture Decisions

- Document major architectural decisions in `/docs/`
- Use ADR (Architecture Decision Records) format
- Explain context, decision, and consequences
- Keep decisions dated and versioned
